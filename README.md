#Суть
 Код и технология нужны для того что бы:
 **При загрузке ПО, игр, дистрибутивов ОС:** Убедиться, что скачанный образ не содержит ошибок.
 
-**При передаче важных документов** (договоры, презентации, отчеты): Гарантировать, что получатель видит именно то, что вы отправили.

 **При резервном копировании:** Проверить, что архив или копия не повреждены.
 
 **В цифровой криминалистике и юриспруденции:** Доказать, что файл (например, доказательство) не изменялся с момента получения.


#Первое_открытие 
При открытии кода вы увидите главное меню:
=================================================
ПРОГРАММА ПРОВЕРКИ ЦЕЛОСТНОСТИ ФАЙЛОВ
==================================================
1. Создать контрольные суммы для директории
2. Проверить целостность файлов
3. Проверить один файл
4. Выход
--------------------------------------------------
#Первый_шаг
1. Выберите пункт "1" в меню
2. Введите путь к директории: /путь/к/вашему/проекту
3. Введите имя файла для сохранения (по умолчанию: checksums.json)
4. Выберите алгоритм хеширования:
   - 1. MD5 (быстрый, но менее безопасный)
   - 2. SHA-1 (устаревший)
   - 3. SHA-256 (рекомендуется)
   - Выбираем 3
**Пример работы:**

Сканирование директории: /home/user/project
--------------------------------------------------
 Обработан: README.md
 Обработан: src/main.py
 Обработан: data/config.json
 Обработан: images/logo.png
--------------------------------------------------
Готово! Обработано файлов: 4
Контрольные суммы сохранены в: checksums.json

#Второй_шаг

### Проверка целостности файлов
1. Выберите пункт "2" в меню
2. Введите путь к файлу с контрольными суммами: checksums.json
3. Введите путь к директории для проверки:
   - Enter - использовать путь из файла checksums.json
   - Или укажите новый путь: /новый/путь/к/файлам
**Пример результата проверки:**
Проверка целостности файлов в: /home/user/project
Используется алгоритм: sha256
--------------------------------------------------
 Целостность подтверждена: README.md
 НАРУШЕНА ЦЕЛОСТНОСТЬ: src/main.py
  Ожидалось: a1b2c3...
  Получено:  d4e5f6...
 Файл отсутствует: data/config.json
--------------------------------------------------
РЕЗУЛЬТАТЫ ПРОВЕРКИ:
Всего файлов для проверки: 3
 Целостность подтверждена: 1
 Нарушена целостность: 1
 Отсутствуют файлы: 1
#Третий_шаг
### Проверка одного файла
1. Выберите пункт "3" в меню
2. Введите путь к файлу: /путь/к/файлу/установщик.exe
3. Выберите алгоритм (1-3)
4. Получите хеш-сумму
5. По желанию сравните с известной хеш-суммой

**Пример вывода:**
==================================================
Файл: /downloads/ubuntu-22.04.iso
Размер: 3670016000 байт (3.5 GB)
Алгоритм: SHA-256
Хеш-сумма: a1b2c3d4e5f67890123456789012345678901234567890123456789012345678
==================================================

Сравнить с известной хеш-суммой? (y/n): y
Введите известную хеш-сумму: a1b2c3d4e5f67890123456789012345678901234567890123456789012345678
 Хеш-суммы совпадают! Файл цел.
